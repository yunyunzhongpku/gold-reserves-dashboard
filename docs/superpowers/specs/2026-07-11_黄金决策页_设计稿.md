# 黄金决策页改版设计稿

设计日期：2026-07-11

状态：用户已确认页面方向、交互合同与工程边界，本稿作为实现依据

适用页面：`黄金数据驱动跟踪`

本稿在页面信息架构、姿态计分和中国官方储备源优先级上，取代
`2026-06-25-gold-dashboard-automation-design.md` 中与之冲突的旧设计；
旧稿的数据刷新架构和统计局限说明继续有效。

## 1. 这次改版在解决什么

当前页面已经具备黄金价格、宏观变量、官方储备、ETF、CFTC、风险代理和关系检验，
但信息组织仍像一份全部展开的研究底稿：黄金价格只是八张卡片之一，八组关系检验、
双轴图、滚动相关和阶段表默认铺开，用户难以快速回答三个最重要的问题：

1. 黄金现在是什么趋势？
2. 当前哪些力量在支持或压制黄金？
3. 最近最值得注意的变化是什么？

同时，中国央行购金图存在真实的数据拼接错误。当前
`data/market/official_reserves_manual.csv` 只覆盖 2026 年 5、6 月，
却接在滞后的 Excel 历史序列之后，导致页面把 5 月增量算成约 `+23.04 吨`，
再把 6 月 `+14.93 吨` 画成低于上月的柱子。

国家外汇管理局官方表给出的序列是：2025 年 12 月 7415 万盎司，
2026 年 1—6 月依次为 7419、7422、7438、7464、7496、7544 万盎司。
按 `1 万金衡盎司 = 0.311034768 吨` 换算：

| 月份 | 官方储备（万盎司） | 换算储备（吨） | 环比增量（吨） |
|---|---:|---:|---:|
| 2025-12 | 7415 | 2306.323 | 锚点 |
| 2026-01 | 7419 | 2307.567 | +1.244 |
| 2026-02 | 7422 | 2308.500 | +0.933 |
| 2026-03 | 7438 | 2313.477 | +4.977 |
| 2026-04 | 7464 | 2321.564 | +8.087 |
| 2026-05 | 7496 | 2331.517 | +9.953 |
| 2026-06 | 7544 | 2346.446 | +14.930 |

所以正确图形必须表现为：6 月柱高于 5 月柱，且两者都位于零轴上方。

官方来源：

- [国家外汇管理局：官方储备资产（2025）](https://www.safe.gov.cn/safe/2025/0206/27115.html)
- [国家外汇管理局：官方储备资产（2026）](https://www.safe.gov.cn/safe/2026/0206/27116.html)

## 2. 目标与范围

### 本轮目标

- 把首页改成 1—2 分钟可读完的黄金决策页。
- 首屏只回答“金价与趋势、当前主要驱动、最近发生了什么”。
- 修正中国央行储备历史序列和环比柱状图。
- 为黄金主图增加区间、均线显隐和悬浮读数交互。
- 将关系检验、历史阶段、数据质量和方法说明收进默认折叠的“研究依据”。
- 保持现有静态构建和 GitHub Pages 发布边界，不增加运行时服务。

### 本轮不做

- 不引入 React、Chart.js、ECharts 或其它前端依赖。
- 不重写整个 `scripts/build_site.py`，不做与页面改版无关的文件拆分。
- 不改变 Wind/FRED/CFTC 的刷新入口和 GitHub Pages 发布链路。
- 不新增预测模型、交易信号、仓位建议或复杂权重优化。
- 不在本轮自动抓取国家外汇管理局网页；官方序列仍由本地 CSV 明确维护。

## 3. 已确认的设计决策

| 编号 | 决策 | 理由 |
|---|---|---|
| D1 | 首页采用“决策页 + 折叠研究” | 优先服务日常快速判断，同时保留可追溯研究证据。 |
| D2 | 黄金价格与技术状态成为首屏主轴 | 黄金自身表现是所有驱动判断的落点，不应继续作为普通因子卡。 |
| D3 | 首屏姿态只由五组独立驱动形成 | 减少重复计数和弱代理变量制造的过度精确感。 |
| D4 | 价格趋势不参与驱动合成 | 它是标的自身派生信号，不是独立外生驱动。 |
| D5 | EPU、GPR 不参与驱动合成 | 两者是间歇有效的避险代理，保留研究价值，但不应左右首页姿态。 |
| D6 | 最近变化固定回答三个主题 | 宏观、官方购金、资金与仓位比机械罗列所有指标更稳定、更易读。 |
| D7 | Python 负责计算，JavaScript 只负责展示交互 | 保证状态与口径只有一套，静态页面也能复核。 |
| D8 | 中国央行数据以国家外汇管理局完整月序列为准 | 当前 Excel 历史存在月份错位，官方原始披露优先。 |
| D9 | 页面无 JavaScript 时仍显示默认一年图 | 交互是渐进增强，不以空白页面换取动效。 |
| D10 | 不引入新依赖 | 当前原生 HTML、SVG、JavaScript 足以完成所需交互。 |

## 4. 页面结构

页面从上到下只有五个主要区域。

### 4.1 顶栏

显示：

- 标题“黄金数据驱动跟踪”；
- 主要市场数据的最新日期；各驱动仍单独显示自己的观测日和质量；
- 五组驱动的数据可用数和过期数；
- 驱动合成姿态，例如“承压”；
- 一句构成说明，例如“3 项压力、1 项支持、1 项中性”。

首屏不再显示英文 `score / tendency` 作为主文案。原始分、有效分母和归一化倾向
放进“研究依据”的方法区，供复核而非抢占注意力。

### 4.2 黄金价格与技术状态

左侧为黄金价格主图，右侧为技术状态摘要。

主图包含：

- 当前金价、最近 5 个有效观测日收益；
- `3个月 / 1年 / 3年` 三个区间按钮，默认 `1年`；
- `MA20 / MA60 / MA200` 三个独立显隐按钮，默认全部显示；
- 鼠标或触摸移动时显示日期、金价和三条均线值；
- 价格、MA20、MA60、MA200 使用固定且可区分的颜色与线型；
- 桌面端提供竖向指示线，移动端点击后固定读数。

右侧技术摘要显示：

- 短期状态；
- 中期状态；
- 是否形成严格多头/空头排列；
- 当前价格相对三条均线的百分比距离；
- 一条明确的判断改变条件。

### 4.3 当前主要驱动

使用紧凑表格，不再使用八张等权大卡。每行只保留：

- 状态色点；
- 驱动名称和所属主题；
- 最新值；
- 最近变化；
- 一句含义；
- 数据质量。

默认顺序固定为：实际利率、美元、中国央行购金、通胀预期、黄金 ETF、CFTC 净多。
ETF 与 CFTC 在界面上分两行，合成姿态中只占“资金与仓位”一票，避免资金主题重复计权。

### 4.4 最近发生了什么

始终只生成三条主题化摘要：

1. **宏观条件**：实际利率与美元近月变化是否同向支持或压制黄金；
2. **官方购金**：中国当月增量与上月增量是加速、放缓还是转为净卖出；
3. **资金与仓位**：ETF 与 CFTC 是同向还是分化。

这三条由确定性规则生成，不调用 LLM，不做跨量纲“最大变化”排序。

### 4.5 研究依据

使用原生 `<details>`，默认关闭。展开后依次包含：

- 因子关系检验；
- 央行购金历史图；
- 阶段表现表；
- EPU、GPR 和长期估值等辅助观察；
- 数据质量与来源；
- 计算方法、阈值、已知局限。

现有关系计算保留，不在本轮重写统计方法；只改变默认呈现层级。

## 5. 黄金价格和技术判定合同

### 5.1 输入与均线

- 输入为按日期升序、`gold_price` 非空的日频有效观测。
- `MA20 / MA60 / MA200` 均为对应窗口的简单移动平均。
- 均线必须基于完整历史计算，再按所选显示区间裁剪；切换区间不得重新以区间起点计算均线。
- 不足 20、60 或 200 个有效观测时，对应均线为 `None`，界面显示“样本不足”并禁用该开关。

### 5.2 严格排列

- **多头排列**：`price >= MA20 > MA60 > MA200`。
- **空头排列**：`price <= MA20 < MA60 < MA200`。
- 其它情况统一为“未形成完整排列”，不得仅凭价格高于或低于 MA200 就称为多头或空头排列。

### 5.3 短期与中期状态

- **短期反弹**：价格高于 MA20，且最近 5 个有效观测日收益为正。
- **短期走弱**：价格低于 MA20，且最近 5 个有效观测日收益为负。
- 其它情况为“短期震荡”。
- **中期偏多**：价格同时高于 MA60 与 MA200。
- **中期偏空**：价格同时低于 MA60 与 MA200。
- 价格夹在 MA60 与 MA200 之间时为“中期修复”。

按当前数据，金价 `4174.19`，MA20 `4156.72`、MA60 `4472.52`、MA200 `4420.02`，
最近 5 个有效观测日收益约 `+3.9%`，因此显示：
“中期偏空 · 短期反弹 · 未形成完整空头排列”。

### 5.4 图表交互

- 区间按最新观测日向前计算日历边界，不用固定行数代替自然月或自然年。
- 图表初始 HTML 包含服务器生成的一年静态 SVG。
- Python 将日期、金价和三条均线序列以 JSON 嵌入
  `<script type="application/json" id="gold-chart-data">`；序列化时转义 `<`，防止脚本标签提前闭合。
- JavaScript 读取 JSON 后增强同一图表，不重新计算指标。
- 区间按钮和均线按钮使用真实 `<button>`，同步维护 `aria-pressed`，支持键盘操作。
- tooltip 不遮挡最新值；离开图表时恢复到最新一个有效观测。

## 6. 五组驱动与姿态合同

### 6.1 五组驱动

| 驱动组 | 输入 | 状态规则 |
|---|---|---|
| 实际利率 | FRED DFII10 | 近 21 个有效观测变化 `< -0.05pct` 为支持，`> +0.05pct` 为压力，否则中性。 |
| 美元 | Wind USDX.FX | 近 21 个有效观测变化 `< -0.5` 为支持，`> +0.5` 为压力，否则中性。 |
| 通胀预期 | FRED T10YIE | 近 21 个有效观测变化 `> +0.05pct` 为支持，`< -0.05pct` 为压力，否则中性。 |
| 官方购金 | 中国官方月度黄金储备 | 当月净增为支持、净减为压力、零变化为中性；全球储备只作背景，不再因缺失把中国状态降为中性。 |
| 资金与仓位 | ETF 近月变化 + CFTC Managed Money 近 4 周变化/拥挤度 | 两个可用子信号分别投 `+1/0/-1`，平均值 `>= +0.5` 为支持，`<= -0.5` 为压力，其余中性。 |

资金与仓位中：

- ETF 变化超过 `+3 吨 / -3 吨` 才分别投支持/压力；
- CFTC 三年分位 `> 90%` 投压力；否则近 4 周净多变化 `> +15,000` 投支持，
  `< -15,000` 投压力，其余中性；
- GVZ 继续作为风险和拥挤背景展示，不参与方向投票；
- 金价动量从资金与仓位组移除，避免与首屏技术状态重复计数。

### 6.2 合成姿态

- 状态权重：支持 `+1`，压力 `-1`，中性 `0`。
- `missing` 和 `very-stale` 不进入分子或分母；`stale` 保留计分但明确标记。
- `tendency = 净分 / 有效驱动组数`。
- `tendency >= +0.25` 为“偏多”；`tendency <= -0.25` 为“承压”；其余为“中性”。
- 若有效驱动少于 3 组，不给合成姿态，显示“数据不足”。

价格技术状态与驱动姿态并列展示，不合成一个新的总分。例如：
“驱动承压；技术中期偏空、短期反弹”。

## 7. 中国央行数据修正合同

### 7.1 本地官方覆盖文件

继续沿用 `data/market/official_reserves_manual.csv`，但把中国官方原始披露值作为输入，
列为：

```text
date,china_reserves_10k_oz,global_reserves,source
```

- 从 2025-12 锚点开始，补齐至最新官方月份；
- `china_reserves` 在 Python 中按 `万盎司 × 0.311034768` 派生，不在 CSV 中重复保存换算值；
- `source` 使用国家外汇管理局官方表，不再写“Wind 通讯社”；
- 展示保留两位小数，计算保留完整浮点精度。

### 7.2 字段级合并

现有 `merge_override_rows()` 会用整行覆盖，导致中国手工更新同时抹掉同月可用的全球储备。
本轮改为储备专用的字段级合并：

- 中国字段和中国来源由官方覆盖文件替换；
- 全球字段若覆盖文件为空，则保留 Excel 中同月的全球值和来源；
- 新月份不存在 Excel 行时，全球字段保持缺失；
- UI 分开显示中国来源和全球来源。

### 7.3 连续性与失败行为

覆盖文件必须满足：

- 日期严格升序且月份唯一；
- 从首个锚点到最新月份逐月连续；
- 至少有前一个月锚点，才能展示最新月环比；
- 中国原始值必须为正数；
- `date > today` 的月度行不得进入最新值、环比或图表；
- 同月重复、月份断档、日期倒序或非法数值均抛出明确错误，停止生成页面。

这次已通过官方重叠区间核对确认 Excel 历史错位，因此国家外汇管理局序列被明确指定为
中国字段的权威来源；覆盖不是静默容忍不一致，而是经过验证后的源优先级决策。

## 8. 数据流与代码边界

现有发布架构保持不变：

```text
本地刷新 Wind / FRED / CFTC
  → 提交 CSV 与必要的 Excel
  → scripts/build_site.py 计算数据、状态、图表序列
  → site/index.html
  → GitHub Pages 发布静态文件
```

`scripts/build_site.py` 内增加少量职责明确的辅助函数：

- 读取与校验中国官方覆盖序列；
- 字段级合并储备数据；
- 计算 MA20/MA60/MA200 和技术状态；
- 选择五组驱动并计算姿态；
- 生成三条主题化最近变化；
- 序列化黄金交互图数据；
- 渲染首屏和折叠研究区。

不拆文件、不改变刷新脚本公共入口。JavaScript 以内联、无依赖方式写入生成 HTML，
只负责图表视图状态。

## 9. 降级、可访问性与响应式

- 数据缺失：显示“缺失”，不生成伪值，不参与姿态。
- 数据很旧：显示“数据过旧，未计入姿态”，但仍可在研究依据中查看历史值。
- 均线样本不足：禁用对应按钮，技术判断只使用可用部分；无法形成中期判断时写“样本不足”。
- 全球储备未更新：只写“本期未更新”，不推断全球央行净买入或净卖出。
- JavaScript 失败：默认一年静态图、首屏数值、驱动表和研究内容仍可阅读。
- 所有状态不只依赖颜色，同时显示“支持 / 中性 / 压力”文字。
- 图表具有 `role="img"` 和可读 `aria-label`；按钮可键盘操作。
- 桌面端在约 `1440 × 900` 下完成核心判断；窄屏按“价格 → 技术 → 驱动 → 最近变化”自然堆叠，
  不出现必须横向滚动才能读完的首屏表格。

## 10. 测试与验收

### 10.1 自动测试

沿用 `unittest`，优先测试先行，新增或更新以下行为测试：

1. 官方 2025-12 至 2026-06 序列可读、连续且换算正确；
2. 2026-05 增量约 `+9.95 吨`，2026-06 增量约 `+14.93 吨`，且 6 月大于 5 月；
3. 重复月份、断档、倒序和非法值导致构建失败，未来日期不会进入页面；
4. 字段级合并不会因中国覆盖而抹掉同月全球储备；
5. 多头排列、空头排列、未形成完整排列的边界；
6. 短期反弹/走弱/震荡和中期偏多/偏空/修复；
7. 五组驱动的有效分母、`missing`、`very-stale` 和少于 3 组时的行为；
8. 资金与仓位不再把金价动量计入投票；
9. 最近变化始终为宏观、官方购金、资金与仓位三条；
10. HTML 包含区间和均线按钮、图表 JSON、折叠研究区，且不再默认展开关系表；
11. HTML 不再出现冗长的 `What Changed / 失效条件 / 下一观察点` 三列重复清单；
12. 原有数据来源、未来日期过滤、关系检验和前瞻收益守护测试继续通过。

### 10.2 浏览器验收

- `3个月 / 1年 / 3年` 切换改变可见区间；
- 三条均线可独立显隐，按钮状态与实际线条一致；
- 鼠标悬浮和移动端点击显示正确日期与数值；
- “研究依据”默认关闭且可正常展开；
- 桌面和移动端无溢出、遮挡或不可读文本；
- 控制台没有错误；
- JavaScript 被禁用时仍有默认图表和完整文字信息。

### 10.3 完成前验证命令

```bash
python3 -m unittest tests.test_build_site tests.test_refresh_wind_data tests.test_update_data_and_commit -v
python3 scripts/build_site.py
git diff --check
```

完成实现不等于发布。本轮在测试与浏览器验收通过后先停在本地工作区；
`git push` 和 GitHub Pages 部署仍需用户单独确认。

## 11. 实现工作流划分

实现阶段可以并行做三条边界清晰的工作流，最终由主代理合并与验收：

1. **数据正确性**：官方储备 CSV、读取/连续性校验、字段级合并及测试；
2. **页面与交互**：技术指标、五组驱动、首屏 HTML/CSS/JavaScript 及测试；
3. **独立验收**：复核口径、测试覆盖、浏览器桌面/移动端和无 JavaScript 降级。

数据结构和公共函数签名由主代理先定，避免并行修改同一段代码。正式实现仍以小步、可逆、
可验证为原则，不做与本设计无关的清理。

## 12. 已知局限

- 五组驱动仍是等权启发式框架，不代表统计最优权重。
- 实际利率、美元和通胀预期仍存在共线性；本轮通过减少其它重复层降低问题，但不做因子正交化。
- EPU、GPR 和关系检验仍依赖现有数据与阈值，其结论只作为研究证据，不作为首页姿态。
- 国家外汇管理局序列仍需人工追加；自动抓取和更新属于后续独立任务。
- 现有滚动相关使用重叠窗口，显著性限制继续沿用旧设计中的已知局限说明。
