状态:用户已确认信息架构、两大核心视图与全部数据口径决策;2026-07-13 评审修订三处合同
(撤回自动分段、复合 summary 改计数口径、钩子条合同细化)并收缩首版范围;其余前端细节经用户明确授权由本稿代定。本稿作为实现依据。

## 1. 这次改版在解决什么

7-11 改版把研究内容整体降级折叠,解决了「别挡首屏的路」,但没有解决「展开后如何高效阅读」。
现状四个问题(与用户确认的不满意维度一一对应):

1. **信息组织**:1305 行内容装在一个 `<details>` 里,8 张因子卡六段式纵向平铺,无总览、无导航;
2. **图表形式**:20 张 SVG 全部无中间刻度、无网格,双轴图有刻度伸缩的固有歧义;
3. **洞察层次**:只有「单因子 vs 金价」逐个陈列,关系的时间演变要靠人眼对照阶段表(且阶段边界有 4 套互不一致的硬编码);
4. **与首屏脱节**:首屏五驱动与研究区八因子是两套叙事,驱动行无法下钻到对应证据。

改版后研究区要快速回答两个问题(用户选定,排除了收益归因与独立总览热图):

1. **首屏判断溯源**:首屏说某驱动是支持/压力,点进来能看到完整证据链;
2. **关系结构演变**:每个因子与金价的关系在时间上怎么变化,现在还靠不靠谱。

## 2. 目标与范围

### 本轮目标(7-13 收缩后)

- 研究区从「一个大折叠」重组为「多单元折叠」:五驱动证据单元 + 辅助观察 + 方法与数据,共 7 个独立 `<details>`;
- 首屏驱动表每行加「证据」锚链,点击跳转并自动展开对应单元;
- 每个证据单元内:溯源钩子条(含阈值与判定推导)→ 一句读法 → 反转轴主关系图 → 滚动相关折线 + 三态阈值参考带;
- 全部研究区图表补齐:中间刻度、淡网格、末点数值;购金柱状图补 Y 轴刻度;
- 阶段表退役,4 套手工阶段边界随之退役;关系演变由「滚动相关折线落在哪个阈值带」直读;
- 折叠态的 summary 行自带轻量总览:单因子单元「有效性徽章 + 滚动相关 sparkline + 当前相关值 + 新鲜度」;复合单元(资金与仓位)只显示「N 有效 · N 偏弱 · N 失效」子块计数,不显示任何合成相关值。

### 本轮不做

- 不做收益归因模型(GRAM 式回归分解)、不做滚动 beta,不引入任何统计新方法;
- 不做「因子 × 时间」独立总览热图(summary 行已承担轻量总览职责);
- **不做自动 regime 分段与色条**(7-13 评审撤回:最小段长合并会让段标签与段均值口径矛盾,见 §13;若阈值带方案在浏览器验收后仍难读,再单独立项讨论);
- **不做通用悬浮读数 JS**(7-13 收缩暂缓:读数由轴刻度 + 末点数值静态承接;原 E14 采样合同封存备用);
- **不产出任何「综合相关」**:复合指标(资金与仓位)三个子块口径不同(ETF/CFTC 带方向,GVZ 看绝对强弱),带符号相关系数直接平均没有可解释含义,现有 `latest_corr` 平均逻辑随本轮退役;
- 不改首屏五驱动的状态规则、姿态合成、最近变化生成逻辑;
- 不改数据刷新链路与发布链路,不引入前端依赖,不拆分文件部署;
- 不改关系检验的统计计算本身(变化对构造、滚动窗口、前瞻收益守护均沿用)。

## 4. 研究区结构

研究区标题「研究依据」升级为常显区块标题(不再是大 `<details>` 的 summary),其下 7 个独立 `<details>`:

| 顺序 | 单元 | id 锚点 | 内容 |
|---|---|---|---|
| 1 | 实际利率 | `evidence-real-rate` | 标准证据单元 |
| 2 | 美元 | `evidence-dollar` | 标准证据单元 |
| 3 | 官方购金 | `evidence-reserve` | 标准证据单元 + 并入现「央行购金历史」的中国/全球环比柱状图 |
| 4 | 通胀预期 | `evidence-inflation` | 标准证据单元 |
| 5 | 资金与仓位 | `evidence-positioning` | 复合单元:ETF、CFTC、GVZ 三个子块 |
| 6 | 辅助观察 | `evidence-auxiliary` | 价格与趋势、EPU、GPR 三个子块 + 长期估值快照;顶部一句提示「以下不参与首页姿态」 |
| 7 | 方法与数据 | `evidence-method` | 现「数据质量与来源」表 + 「计算方法与已知局限」合并 |

顺序 1–5 与首屏驱动表一致。现研究区四大板块的去向:因子关系检验 → 单元 1–6;央行购金历史 → 并入单元 3;数据质量与来源、计算方法与已知局限 → 单元 7。无内容删失,只有重组与消重。

## 5. 证据单元合同

### 5.1 summary 行(折叠态可见)

**单因子单元(1–4)**,从左到右:开合指示符、因子名 + 来源短注、有效性徽章、滚动相关 sparkline(约 64×16,画判定用滚动相关线近 2 年)、「滚动相关 +0.41」当前值(标签统一用「滚动相关」,各因子具体窗口在方法节列明)、数据新鲜度点(右对齐,沿用 `classify_staleness` 结果)。

- 徽章 = `corr_tone(判定用滚动相关最新值, expected)`,词表「有效/偏弱/失效」,色 = 现有 supportive/neutral/headwind 系;
- 徽章语义是「历史关系靠不靠谱」,与首屏「支持/压力」(现在什么方向)分工,两者并存不合成。

**复合单元(资金与仓位)**:徽章位显示三子块 `corr_tone` 计数文字,如「2 有效 · 1 失效」;计数覆盖「有效/偏弱/失效/样本不足」四种输出,为零的不写——**子块样本不足必须显式计入,不得静默省略**(缺失显式化红线);**无 sparkline、无合成相关值**(E16)。各子块的相关值与徽章在展开后的子块标题处分项显示。

**辅助观察单元**:徽章固定「不计分」中性样式,无 sparkline、无相关值;**方法与数据单元**:无徽章、无 sparkline。

### 5.2 展开态结构(自上而下)

1. **溯源钩子条**(淡黄底,仅单元 1–5),合同(E15):
   - 计入姿态时,正文 = 「首屏当前:{状态徽章} —— {观测窗变化} {容忍带阈值与方向规则} → 判『{状态}』。」示例:「首屏当前:压力 —— 近 21 个有效观测变化 +0.18pct,容忍带 ±0.05pct,利率下行超带判支持、上行超带判压力 → 判『压力』。」
   - 变化值取 `layer["change"]`/`change_label`,阈值与窗口引用现有模块常量(`REAL_RATE_CHANGE_THRESHOLD` 等六驱动各自常量),不新增口径、不硬编码数字;
   - 复合单元(资金与仓位)钩子条分三行:ETF 子判定一行(±3 吨容忍带)、CFTC 子判定一行(±15,000 张容忍带 + 三年分位 >90% 拥挤直接判压力)、合成规则一行(两票均值 ≥+0.5 支持 / ≤−0.5 压力,GVZ 不投票);
   - `data_quality` ∈ {missing, very-stale} 或 state 不在三态时,正文 = 「数据{缺失/严重滞后}(滞后 N 天),该驱动本期未计入首屏姿态。」——与 `score_driver_layers` 的真实剔除行为一致(它剔除 missing/very-stale 层;不足 3 层时整体降级「数据不足」);
   - 末尾链接「完整规则见方法与数据」。
2. **一句读法**:说明预期方向的机制一句 + 当前有效性状态一句;明确「首屏判断来自变化规则,不依赖历史相关成立」这类边界(模板化生成,沿用现有文案函数改写)。
3. **主关系图**:见 §6。
4. **关系演变**:滚动相关折线 + 三态阈值参考带,见 §7。
5. **尾注小字**:窗口参数与 n(E9)、「阈值与已知局限见方法与数据」。

复合单元与辅助观察的子块 = 上述 3–4 两段(子块标题 + 主图 + 演变),不重复钩子条(复合单元的钩子条在单元顶部,覆盖三行子判定)。

## 6. 图表合同

### 6.1 反转轴主关系图

- 双轴:左轴金价,右轴因子;**仅 expected="negative" 的因子(实际利率、美元)右轴数值方向反转**,轴题必须含「(已反转)」字样,图例同步标注;
- 反转只作用于几何方向:**右轴刻度文字始终是原始数值**(反转时大值在下),末点数值标注同为原始值——这是暂缓悬浮读数后原始值的静态承接方式;
- expected="positive"(购金、通胀预期、ETF、CFTC)正常双轴;GVZ(expected="absolute")正常双轴,不反转;
- 所有研究区图表统一补:纵轴两端 + 至少 1 个中间刻度、水平淡网格线;末值显示在图例,折线末点画圆点标记(不在图内加数字文字,避免视觉负担);
- 购金环比柱状图补 Y 轴刻度(现状无任何纵轴数值)。

### 6.2 读数方式(本轮静态)

悬浮读数 JS 本轮暂缓(E14 封存)。读数由三处静态元素承接,信息完整性不依赖 JS:

- 轴两端 + 中间刻度(所有图);
- 图例中的末值(折线末点以圆点标记);
- 反转轴因子的刻度与末值均为原始数值(见 §6.1)。

## 7. 三态阈值参考带合同(演变图)

`make_corr_chart` 增加 `expected` 参数;给定时在绘图区铺满高度方向的水平三态底色带,带边界即 `corr_tone` 阈值,精确翻译如下(区间闭开与 `corr_tone` 的 ≥/≤ 一致):

| expected | 有效(淡绿) | 偏弱(淡灰) | 失效(淡红) |
|---|---|---|---|
| positive | [+0.35, +1] | (−0.1, +0.35) | [−1, −0.1] |
| negative | [−1, −0.35] | (−0.35, +0.1) | [+0.1, +1] |
| absolute | [−1, −0.35] ∪ [+0.35, +1] | (−0.35, −0.1) ∪ (+0.1, +0.35) | [−0.1, +0.1] |

- **渲染**:带 = 全宽 `rect`,置于折线与网格之下,fill 用现有三态色、fill-opacity ≤ 0.08;每条带的右内缘放小字态名(「有效」「偏弱」「失效」),absolute 的两条同态带各标一次;不另画阈值虚线(带边界即视觉分隔);
- **判定线**:主线 = 判定用滚动相关(月频因子:现有月度滚动窗;日频因子:中期 252 日滚动窗;仓位子块:52 期滚动窗);短期滚动相关线细虚线叠加,不影响带布局;
- **图注**(E9):写明窗口与 n,如「252 日滚动窗(n=252,重叠样本)」;
- 无自动分段、无最小段长参数、无段统计——历史某段的状态由读者顺线直读;
- 现 `weak_periods_from_rolling` 的「低相关窗口」文案职责由阈值带取代,函数随本轮退役。

## 11. 测试与验收

沿用 `unittest`,新增/改写以下行为测试(先测后写):

## 12. 已知局限

- **反转轴仍属双轴家族**:刻度伸缩的固有歧义保留(Few 2008 等的批评适用),用户知情选择以换取「同向=成立」的直读性;缓解:「已反转」显著标注 + 右轴刻度与末值均为原始数值;
- **阈值继承**:±0.35/∓0.1 沿用现有 `corr_tone`,其不看样本量的局限一并继承(已在 6-25 spec §7 与 backlog 记录,本轮不解决);
- **重叠窗口**:滚动相关的有效样本/显著性问题沿用旧稿说明,图注写明 n 为重叠观测数,不宣称显著性;
- **历史 regime 段叙事本轮缺席**:阈值带只支持顺线直读,不输出段起止日期、段均值、段内金价收益;这是 7-13 评审的知情取舍(换取口径无矛盾)。若浏览器验收发现折线难读,再议自动分段——届时必须先解决「段标签态与段均值态一致」问题(如按段均值重判态并迭代合并),不得回退到本次撤回的实现;
- **悬浮读数暂缓**:逐点历史读数暂不可得,只能读轴刻度与末点;E14 采样合同封存,后续要做时直接复用;
- 连续「因子 × 时间」热图形态在黄金语境无成熟先例(调研确认),本轮以 summary 行替代总览,若未来需要再议。
