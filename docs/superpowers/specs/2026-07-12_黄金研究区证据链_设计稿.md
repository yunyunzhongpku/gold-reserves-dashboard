# 黄金研究区证据链改版设计稿

设计日期:2026-07-12

状态:用户已确认信息架构、两大核心视图与全部数据口径决策;其余前端细节经用户明确授权由本稿代定。本稿作为实现依据。

适用范围:`site/index.html` 中「研究依据」折叠区(现 825–2129 行)及首屏驱动表的最小联动改动。

本稿取代 `2026-07-11_黄金决策页_设计稿.md` §4.5 对研究区的描述;首屏姿态计分、
黄金主图交互、中国官方储备源等其余合同继续有效,本稿不触碰。

## 1. 这次改版在解决什么

7-11 改版把研究内容整体降级折叠,解决了「别挡首屏的路」,但没有解决「展开后如何高效阅读」。
现状四个问题(与用户确认的不满意维度一一对应):

1. **信息组织**:1305 行内容装在一个 `<details>` 里,8 张因子卡六段式纵向平铺,无总览、无导航;
2. **图表形式**:20 张 SVG 全部无中间刻度、无网格、无悬浮读数,折叠区内零 JS;双轴图有刻度伸缩的固有歧义;
3. **洞察层次**:只有「单因子 vs 金价」逐个陈列,关系的时间演变要靠人眼对照阶段表(且阶段边界有 4 套互不一致的硬编码);
4. **与首屏脱节**:首屏五驱动与研究区八因子是两套叙事,驱动行无法下钻到对应证据。

改版后研究区要快速回答两个问题(用户选定,排除了收益归因与独立总览热图):

1. **首屏判断溯源**:首屏说某驱动是支持/压力,点进来能看到完整证据链;
2. **关系结构演变**:每个因子与金价的关系在时间上怎么变化,现在还靠不靠谱。

## 2. 目标与范围

### 本轮目标

- 研究区从「一个大折叠」重组为「多单元折叠」:五驱动证据单元 + 辅助观察 + 方法与数据,共 7 个独立 `<details>`;
- 首屏驱动表每行加「证据」锚链,点击跳转并自动展开对应单元;
- 每个证据单元内:溯源钩子条 → 一句读法 → 反转轴主关系图 → 色带滚动相关图 + regime 色条;
- 全部研究区图表补齐:中间刻度、淡网格、悬浮读数(触屏为点击定位);
- 阶段表退役,由自动分段的 regime 色条取代;4 套手工阶段边界随之退役;
- 折叠态的 summary 行自带「有效性徽章 + 滚动相关 sparkline + 当前中期相关值 + 新鲜度」,全收起时研究区即一张总览表。

### 本轮不做

- 不做收益归因模型(GRAM 式回归分解)、不做滚动 beta,不引入任何统计新方法;
- 不做「因子 × 时间」独立总览热图(summary 行已承担轻量总览职责);
- 不改首屏五驱动的状态规则、姿态合成、最近变化生成逻辑;
- 不改数据刷新链路与发布链路,不引入前端依赖,不拆分文件部署;
- 不改关系检验的统计计算本身(变化对构造、滚动窗口、前瞻收益守护均沿用)。

## 4. 研究区结构

研究区标题「研究依据」升级为常显区块标题(不再是大 `<details>` 的 summary),其下 7 个独立 `<details>`:

| 顺序 | 单元 | id 锚点 | 内容 |
|---|---|---|---|
| 1 | 实际利率 | `evidence-real-rate` | 标准证据单元 |
| 2 | 美元 | `evidence-dollar` | 标准证据单元 |
| 3 | 官方购金 | `evidence-reserve` | 标准证据单元 + 并入现「央行购金历史」的中国/全球环比柱状图 |
| 4 | 通胀预期 | `evidence-inflation` | 标准证据单元 |
| 5 | 资金与仓位 | `evidence-positioning` | 复合单元:ETF、CFTC、GVZ 三个子块 |
| 6 | 辅助观察 | `evidence-auxiliary` | 价格与趋势、EPU、GPR 三个子块 + 长期估值快照;顶部一句提示「以下不参与首页姿态」 |
| 7 | 方法与数据 | `evidence-method` | 现「数据质量与来源」表 + 「计算方法与已知局限」合并 |

顺序 1–5 与首屏驱动表一致。现研究区四大板块的去向:因子关系检验 → 单元 1–6;央行购金历史 → 并入单元 3;数据质量与来源、计算方法与已知局限 → 单元 7。无内容删失,只有重组与消重。

## 5. 证据单元合同

### 5.1 summary 行(折叠态可见)

从左到右:开合指示符、因子名 + 来源短注、有效性徽章、滚动相关 sparkline(约 64×16,画判定用滚动相关线近 2 年)、「滚动相关 +0.41」当前值(标签统一用「滚动相关」,各因子具体窗口在方法节列明)、数据新鲜度点(右对齐,沿用 `classify_staleness` 结果)。

- 徽章 = `corr_tone(判定用滚动相关最新值, expected)`,词表「有效/偏弱/失效」,色 = 现有 supportive/neutral/headwind 系;
- 徽章语义是「历史关系靠不靠谱」,与首屏「支持/压力」(现在什么方向)分工,两者并存不合成;
- 复合单元(资金与仓位)徽章取三子块中期相关的 `corr_tone` 多数态,平票取「偏弱」;sparkline 画 ETF 子块(首屏投票主信号);
- 辅助观察单元徽章固定「不计分」中性样式;方法与数据单元无徽章、无 sparkline。

### 5.2 展开态结构(自上而下)

1. **溯源钩子条**(淡黄底,仅单元 1–5):「首屏当前:压力 —— 近 21 个有效观测变化 +0.08pct,超过 +0.05pct 阈值 → 判『压力』。」数值与阈值从对应 layer 与既有常量取,不新增口径;末尾链接「完整规则见方法与数据」。数据 `missing`/`very-stale` 时钩子条如实显示「未计入姿态」及原因。
2. **一句读法**:说明预期方向的机制一句 + 当前有效性状态一句;明确「首屏判断来自变化规则,不依赖历史相关成立」这类边界(模板化生成,沿用现有文案函数改写)。
3. **主关系图**:见 §6。
4. **关系演变**:色带滚动相关图 + regime 色条,见 §7。
5. **尾注小字**:窗口参数、「阈值与已知局限见方法与数据」。

复合单元与辅助观察的子块 = 上述 3–4 两段(子块标题 + 主图 + 演变),不重复钩子条。

## 6. 图表合同

### 6.1 反转轴主关系图

- 双轴:左轴金价,右轴因子;**仅 expected="negative" 的因子(实际利率、美元)右轴数值方向反转**,轴题必须含「(已反转)」字样,图例同步标注;
- expected="positive"(购金、通胀预期、ETF、CFTC)正常双轴;GVZ(expected="absolute")正常双轴,不反转;
- 所有研究区图表统一补:纵轴两端 + 至少 1 个中间刻度、水平淡网格线、末点数值;
- 购金环比柱状图补 Y 轴刻度(现状无任何纵轴数值)。

### 6.2 悬浮读数(桌面 hover / 触屏点击定位)

- 复用首屏金价主图的交互模式,泛化为通用函数:容器标 `data-hover-chart`,每图配一个 `<script type="application/json">`(序列化转义 `<`),含采样后的日期数组与各序列数值数组;
- 读数框显示:日期、各序列名称与原始值(反转轴因子显示**原始值**,不显示反转后的视觉值);
- 采样合同(E14):JSON 点位与 SVG path 点位一一对应;
- 无 JS 时:图表静态完整可读,读数功能缺失不影响信息完整性。

## 7. 自动分段合同(色带与 regime 色条)

新增纯函数 `segment_rolling_corr(rolling, expected, min_len)`:

- **输入**:滚动相关序列(月频因子:现有月度滚动窗;日频因子:中期 252 日滚动窗;仓位子块:各自现有窗口),`expected` 沿用各因子现值;
- **逐点状态**:`corr_tone(corr, expected)` → 有效/偏弱/失效;`corr=None`(窗口未满)不进入任何段,色带留白;
- **最小段长**:连续同状态不足 `min_len`(月频 6 个观测,日频 63 个观测)的段并入前一段;序列首段不足 `min_len` 时并入后一段;整段序列不足 `min_len` 时输出单段;
- **输出**:`[{start, end, state, corr(段内均值), n(段内观测数), gold_return(段内金价区间收益)}]`;
- **渲染**:色带 = 各段淡底色(绿/灰/红,置于折线与网格之下);色条 = 同一分段的饱和色块,段内文字「-0.62 · n36」,`<title>` 含「起止 · 相关 · 金价涨跌 · n」;段过窄(< 约 40px)时段内文字省略,信息保留在 `<title>`;
- 短期滚动相关线:细虚线叠画于同图,不参与分段;
- 现 `weak_periods_from_rolling` 的「低相关窗口」文案职责由色条取代,函数保留与否属实现细节。

## 8. 首屏联动合同

- 首屏驱动表每行(实际利率、美元、中国央行购金、通胀预期、黄金 ETF、CFTC 净多)行末加「证据」链接;ETF 与 CFTC 两行同指 `#evidence-positioning`;
- 点击行为(JS):打开目标 `<details>`(`open=true`)后随锚点滚动;`scroll-margin-top` 预留顶部空间;
- 无 JS 降级:纯锚点跳转到收起的 summary 行,手动展开,信息不丢失;
- 首屏技术状态摘要本轮不加链接(价格与趋势在辅助观察内,留待后续)。

## 9. 版式、响应式与可访问性

- 展开态内容列 max-width 680px;SVG 图表 max-width 520px、`height:auto`;
- 单元展开后总高目标:标准单元 ≤ 约 1.5 屏(1440×900 基准);
- 移动端(≤560px):summary 行 sparkline 隐藏(徽章 + 当前值保留);色条段内文字只留相关系数;内容列全宽;
- 徽章、色带、色条不只靠颜色:徽章有文字,色条段有数值,色带语义由图例文字给出;
- 图表保持 `role="img"` + 中文 `aria-label`;`<details>`/`<summary>` 原生键盘可操作;
- 沿用现有 CSS 变量与色板,色带用现有三态色的淡化(fill-opacity ≤ 0.12),色条 ≤ 0.6。

## 12. 已知局限

- **反转轴仍属双轴家族**:刻度伸缩的固有歧义保留(Few 2008 等的批评适用),用户知情选择以换取「同向=成立」的直读性;缓解:轴端标注原始数值、悬浮显示原始值、「已反转」显著标注;
- **阈值继承**:±0.35/±0.1 沿用现有 `corr_tone`,其不看样本量的局限一并继承(已在 6-25 spec §7 与 backlog 记录,本轮不解决);
- **重叠窗口**:滚动相关的有效样本/显著性问题沿用旧稿说明,色带只呈现描述统计,不宣称显著性;
- **段边界漂移**:自动分段随数据更新移动是设计特性(诚实反映现实),但对比历史版本页面时段边界不可复用,方法节注明;
- **段内 n 是观测数而非独立样本数**:重叠窗口下有效自由度低于 n,方法节保留一句提示;
- 连续「因子 × 时间」热图形态在黄金语境无成熟先例(调研确认),本轮以 summary 行 + 色条组合替代,若未来需要再议。
