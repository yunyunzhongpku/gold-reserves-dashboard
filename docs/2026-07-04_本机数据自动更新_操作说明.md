# 本机数据自动更新操作说明

日期: 2026-07-04
更新: 2026-07-05

## 当前边界

本方案采用全自动边界:

- 本机定时刷新 Wind/FRED/CFTC 数据。
- 本机构建 `site/index.html`。
- 本机运行现有 `unittest` 和 `git diff --check`。
- 检查通过后自动提交数据更新 commit。
- commit 成功后自动 `git push`,触发 GitHub Pages 构建部署。

GitHub Actions 仍只负责 Pages 构建部署。Wind 是 LAN-only,不能放到 GitHub runner 上抓取。

## 自动任务做什么

定时任务调用:

```bash
/usr/bin/python3 /Users/xu/code/python-lab/gold-reserves-dashboard/scripts/update_data_and_commit.py
```

脚本顺序:

1. 要求 tracked 工作区在启动前是干净的;未跟踪文件不影响运行。
2. 运行 `scripts/refresh_wind_data.py`。
3. 运行 `scripts/refresh_market_data.py`。
4. 运行 `scripts/build_site.py`。
5. 运行 `python3 -m unittest tests.test_build_site tests.test_refresh_wind_data tests.test_update_data_and_commit -v`。
6. 运行 `git diff --check`。
7. 只 stage 下面 5 个文件:

```text
data/market/wind_daily.csv
data/market/fred_t10yie.csv
data/market/fred_dfii10.csv
data/market/cftc_gold_cot.csv
site/index.html
```

8. 如有变化,创建本地 commit。
9. commit 成功后,通过 SSH push 到 GitHub,触发 GitHub Pages 构建部署。

如果刷新后出现其它 tracked 文件改动,脚本会失败并停止。

## 安装定时任务

模板在:

```text
launchd/com.xu.gold-reserves-dashboard.refresh.plist
```

默认时间为每周一 10:00 本机时间。

安装:

```bash
mkdir -p ~/Library/LaunchAgents
cp launchd/com.xu.gold-reserves-dashboard.refresh.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.xu.gold-reserves-dashboard.refresh.plist
```

查看:

```bash
launchctl list | grep gold-reserves-dashboard
```

卸载:

```bash
launchctl unload ~/Library/LaunchAgents/com.xu.gold-reserves-dashboard.refresh.plist
```

## 发布到线上

默认自动任务会在检查通过后 push。`git push` 会触发 `.github/workflows/pages.yml`,由 GitHub Pages 部署最新静态页面。

## 当前线程人工触发约定

在当前 Codex 线程中,当用户说:

```text
刷新数据+commit
```

默认含义是执行完整刷新发布流程:

1. 确认 tracked 工作区没有其它未提交改动。
2. 运行 `python3 scripts/update_data_and_commit.py`。
3. 让脚本完成本机刷新、构建、测试、`git diff --check`、本地 commit 和 push。
4. push 成功后,GitHub Pages 自动构建部署。

如果任一检查失败,停止并向用户报告问题;不要绕过 Wind 对账、不要弱化测试、不要 push。
